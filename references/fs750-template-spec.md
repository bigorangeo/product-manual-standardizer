# FS750 Template Spec

这是 `FS750(n)` 参考说明书的版式特征摘要，用于后续统一其他产品说明书时对齐格式。

## Overall Structure

- 标题区为中文产品名一行、英文产品名一行
- 版本信息单独一行
- 正文默认采用 7 个一级章节
- 每个一级章节固定两行标题：中文一行，英文一行
- 正文大量采用“中文一段 + 英文一段”的紧邻配对形式
- 枚举内容采用“中文列表项 + 英文对应句”的交替形式
- 技术指标集中在单张 5 列表格中
- 页眉包含公司中英文名称与物料编码

补充成稿原则：

- 固定模板不等于机械保留所有章节
- 源文件没有独立出现、且没有内容支撑的模块，可以不在成稿中保留
- `推荐工艺参数` 没有明确来源时应直接省略，不要根据零散参数拼出完整章节
- 但“重要声明”这类风险提示模块，可根据全文已有内容整理补成正式章节
- 省略某章节后，后续章节编号需要自动顺延

## Detected Sections

1. `一、产品描述 / I. Product Description`
2. `二、应用领域 / II. Application Fields`
3. `三、主要技术指标 / III. Key Technical Specifications`
4. `四、推荐工艺参数 / IV. Recommended Process Parameters`
5. `五、使用注意事项 / V. Precautions for Use`
6. `六、包装、储存与运输 / VI. Packaging, Storage & Transportation`
7. `七、质量保证与技术支持 / VII. Quality Assurance & Technical Support`

## Technical Table Pattern

技术指标表固定 5 列：

1. `序号 / No.`
2. `测试项目 / Test Item`
3. `测试条件 / Test Condition`
4. `技术指标 / Specification`
5. `参考标准 / Standard`

排版原则：

- 中文在上，英文在下
- 英文与中文左对齐
- 如果上下两行内容完全相同，只保留一行，不重复显示相同数据
- 单元格内容拥挤时优先缩小字号与行距，不要横向硬挤
- 表头也采用中上英下的双语结构
- `参考标准` 默认只保留标准编号本体，不保留年份

## List Pattern

适合整理成列表的内容包括：

- 核心特性
- 主要应用
- 工艺步骤
- 使用注意事项
- 包装与储存条件

但列表不是必须机械保留。

如果某个列表块与上方总述明显重复，例如“主要应用”只是把上一段重新拆成 bullet，则应优先删除整个重复块，并同步清理残留的项目符号、缩进和空白。

如果某个 bullet 后面已经没有正文内容，则该 bullet 行本身也必须删除，不能留下空黑点。

## Process Section Pattern

推荐工艺参数章节适合拆成小节，例如：

- `1. 工件预处理 / Workpiece Pretreatment`
- `2. 浸渍工艺 / Impregnation Process`
- `3. 固化条件 / Curing Conditions`

## Header Pattern

参考页眉字段：

`广州飞思合成材料有限公司 | Guangzhou Faith Synthetic Materials Co., Ltd. | 物料编码 Material No.: 926w1000`

页眉补充规则：

- `logo` 放在页眉左侧，贴近 `广州` 前面
- 页眉整体优先保持单行连续布局
- 不要把页眉预先分割成左右两块固定空间
- `logo` 与页眉文字的关系以“接近但不遮挡”为准
- 页眉文字可参考 `8pt`
- 颜色参考“黑色，浅色 15%”

## Pagination Pattern

- 章节切换前默认只保留一行必要空白
- 如果表格在当前页放不下，应优先将表格整体移到下一页开始
- 不要出现上一页只剩表头、下一页再继续表格的情况
- 不要出现因为删改内容导致的大块无意义留白

## Typography Pattern

- 主标题保留模板原有字号层级
- 除主标题外，其余章节标题、小标题默认可在模板基础上整体放大 2 号
- 正文默认也可在模板基础上统一放大 2 号，以提高阅读性
- 表格文字不跟随正文和标题放大规则，保持独立控制
- 修订日期、版本行、页眉字段不自动套用正文字号放大规则
- 修订日期默认比正文字号小 2 号

## Usage Note

这个参考文件描述的是“结构和排版逻辑”，不是要求逐字照抄 FS750 的正文内容。

最终目标是：

- 固定模板
- 智能清洗内容
- 输出正式、统一、可交付的说明书成稿
