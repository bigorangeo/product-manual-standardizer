# 产品说明书优化排版

把用户提供的产品说明书内容，整理成统一、可交付、可复用的标准说明书格式。默认参考模板为 `FS750(n)` 水性环氧绝缘树脂产品说明书的版式和结构。

这个 skill 的目标不是机械提取文字，也不是简单“套壳排版”，而是：

- 固定说明书版式
- 智能清洗原始内容
- 优化明显不合理的表达和结构
- 输出真正可交付的正式说明书成稿

核心原则：

1. 版式固定
2. 内容可优化

也就是说，页眉、章节骨架、中英排版关系、技术表结构、分页规则应保持统一；但正文中的空项目、重复项、残留占位、明显不顺的说法、表格拥挤问题，必须主动整理优化，不能原样照搬。

## When To Use

在以下场景使用：

- 用户给出某个产品的说明书，希望“照这个模板整理”
- 用户有旧版 `docx`、`pdf`、图片识别文本、纯文本资料，希望重排成统一说明书
- 用户希望后续多个产品说明书都按同一套版式输出
- 用户要把杂乱的产品参数、工艺条件、包装储运要求，整理成正式产品说明书
- 用户说“按照产品说明书排版要求优化这个文件”
- 用户说“按产品说明书要求优化排版”
- 用户说“按说明书规范优化这个文件”
- 用户说“把这个文件按说明书格式优化一下”

不要在这些场景使用：

- 用户只是要摘要、翻译、提取参数，不要求重排成正式说明书
- 用户要做营销彩页、海报、网页落地页，这不是说明书标准化任务

## Default Output Goal

默认输出为一份正式、统一的中英双语产品说明书，包含：

1. 中文产品名
2. 英文产品名
3. 版本信息
4. 分章节正文
5. 技术指标表
6. 页眉固定信息

如用户明确要求只保留中文，则可以省略英文；否则优先保持中英双语结构。

## Canonical Structure

默认采用以下 7 个主章节：

1. `一、产品描述 / I. Product Description`
2. `二、应用领域 / II. Application Fields`
3. `三、主要技术指标 / III. Key Technical Specifications`
4. `四、推荐工艺参数 / IV. Recommended Process Parameters`
5. `五、使用注意事项 / V. Precautions for Use`
6. `六、包装、储存与运输 / VI. Packaging, Storage & Transportation`
7. `七、质量保证与技术支持 / VII. Quality Assurance & Technical Support`

如果源资料缺少某一章：

- 先尝试从全文中归并相关内容
- 若确实没有，不要编造
- 默认保留该章节并标注 `未提供 / Not Provided`，除非用户要求删除空章节

如果源资料包含额外章节：

- 优先归并到上述 7 个章节中
- 只有在确实无法归并时，才新增小节，而不是随意新增新的一级章节

## Two-Layer Rule

这是本 skill 最重要的工作方式。

### Layer 1: Fixed Layout

以下内容默认固定：

- 页眉结构
- 标题区结构
- 7 章章节骨架
- 中文在前、英文在后的双语逻辑
- 技术表的 5 列结构
- 表格内中上英下的排版方式
- 分页和留白控制规则

### Layer 2: Content Optimization

以下内容必须主动优化，而不是原样照搬：

- 空的项目符号、空编号、空白占位
- 重复的列表块、重复说明
- 明显不顺或不专业的中英文表述
- 明显多余的“质量背书式空话”
- 表格内拥挤、错位、横向硬挤的问题
- 导致表格跨页、只剩表头、段落留大空白的分页问题
- 源文件缺失模块时，判断哪些应直接省略，哪些应根据现有内容补成正式模块

判断标准：

- 如果原内容影响专业性、可读性或版面完整性，就要主动优化
- 如果原内容涉及实质技术信息、合规限制或使用条件，就不要擅自删改

补充判断：

- 如果源文件没有某个独立模块，且全文也没有相关内容支撑，不要在成稿里保留“源文件未提供”式说明
- 如果该模块属于说明书成稿中应保留的合规性或风险提示内容，例如“重要声明”，则应基于全文已有信息整理出正式模块内容，而不是把“缺失说明”直接写进正文

## Formatting Rules

严格模仿参考模板的排版逻辑，而不是字面复制每一行。

### Title Area

- 第一行放中文产品名称
- 第二行放英文产品名称
- 版本信息单独一行，类似：
  `版本 Version V1.0`

如果用户明确要求保留日期，可放在最后一页作为：

- `修订日期：03/2026`
- `Revision Date: 03/2026`

### Section Headings

每个一级章节固定为两行：

- 第一行中文标题
- 第二行英文标题

示例：

```text
一、产品描述
I. Product Description
```

字体层级规则：

- 主标题单独保留模板原有字号层级，不跟随后续规则统一放大
- 除主标题外，其余一级标题、二级标题、双语小标题默认可在模板基础上整体放大 2 号
- 标题放大时，不要联动放大表格文字
- 修订日期不视为正文标题，不参与标题放大规则

### Paragraph Pairing

正文默认按“中文在前，英文在后”成对出现：

- 一个中文自然段后紧跟对应英文段
- 不把中英文混写在同一段里
- 如果英文换行，续行要和上一行英文对齐

正文字号规则：

- 除主标题、其他标题和表格内容外，正文段落默认在模板基础上统一放大 2 号
- 中文正文与英文正文分别保持各自原有中英字号差关系，只做等幅放大
- 版本行、页眉字段不按“正文放大”规则处理，保持独立控制
- 修订日期字号应比正文字号小 2 号，独立控制，不跟随正文或标题同步放大

### Lists

列表项遵循中英成对规则：

```text
- 低 VOC，符合环保要求
  Low VOC, meeting environmental requirements
```

如果原始资料语义上属于“核心特性”“注意事项”“工艺步骤”“主要应用”等枚举信息，应主动整理为列表。

列表清洗规则：

- 删除空 bullet、空编号、空白占位
- 删除内容后必须同步清理残留黑点、缩进和空段落
- 若上文已完整覆盖应用对象，后续重复性的“主要应用”列表应优先整块删除
- 删除列表块后，重新检查分页和段前段后距，避免遗留大块空白
- 如果某个项目符号后面已经没有正文内容，该行连同符号一起删除，不保留空黑点占位

### Subsections

工艺章节允许使用阿拉伯数字小节，例如：

```text
1. 工件预处理
Workpiece Pretreatment
```

常见小节包括：

- 工件预处理
- 浸渍工艺
- 固化条件
- 包装规格
- 储存条件
- 保质期

## Technical Specification Table

技术指标默认整理为 5 列表格：

1. `序号 / No.`
2. `测试项目 / Test Item`
3. `测试条件 / Test Condition`
4. `技术指标 / Specification`
5. `参考标准 / Standard`

处理规则：

- 能进表格的技术指标尽量进表格
- 同一测试项目在不同条件下可拆成多行
- 缺少参考标准时可写 `—`
- 数值、单位、温度、时间范围要统一写法
- 若用户要求隐藏某类具体值，例如 VOC 数值，则保留定性表述，删除具体数值本身
- `参考标准` 默认去掉年份，仅保留标准编号本体，例如 `GB/T 1981.2`

表格排版规则：

- 中文在上，英文在下
- 英文与中文首字左对齐
- 不要把中英文横向并排硬挤在一行
- 如果某个单元格里上下两行内容完全相同，只保留一行，不重复显示
- 内容拥挤时，优先缩小字号、压缩段前段后距和行高
- 出现异常拉大字距、表头被空格撑开时，必须主动重排

## Pagination And Spacing

说明书不是只要“内容都在”就行，分页必须可读。

默认规则：

- 章节切换前不要堆积多个空段落，默认只保留 1 行必要空白
- 若某章标题前出现大块无意义空白，应主动压缩
- 技术表应尽量整块显示，不要被拆得支离破碎
- 如果表格在当前页放不下，应优先把表格整体移到下一页开始
- 不要出现“上一页只剩表头，下一页再接表格”的情况
- 不要出现“上一页是标题，下一页只剩表头”的情况

优先级：

1. 保证表格整体可读
2. 保证章节标题和对应内容关系清晰
3. 再优化留白美观

## Header And Fixed Fields

如果输出 Word 成品，页眉默认采用以下逻辑：

`公司中文名 | Guangzhou Faith Synthetic Materials Co., Ltd. | 物料编码 Material No.: 926w1000`

页眉规则：

- `logo` 默认放在页眉左侧，贴近公司中文名前面
- 目标是“贴近但不遮挡”
- 页眉整体优先保持单行连续布局
- 不要为了放 `logo` 先把页眉切成左右分栏再硬塞内容
- 页眉文字默认参考 `8pt`
- 页眉颜色参考“黑色，浅色 15%”

如用户明确要求特殊 logo 关系，可在不破坏单行可读性的前提下微调，但最终以 Word/WPS 实际效果为准。

## Working Method

接到任务时按以下顺序执行：

1. 读取用户提供的说明书或产品资料
2. 提取产品名称、版本、物料编码、公司信息
3. 提取正文并归类到 7 章结构
4. 整理技术指标表
5. 统一中英配对关系
6. 清洗空项、重复项、残留占位
7. 优化不顺或不专业的表述
8. 调整分页、表格落位和空白行
9. 输出正式成稿

## Content Rules

### Faithful But Improved

忠实于原始资料，但允许为了说明书一致性进行这些优化：

- 调整章节顺序
- 合并重复内容
- 把散文式内容改成列表
- 把零散参数改成表格
- 统一术语和单位
- 删除空项和残留格式
- 精简无信息增量的套话
- 修正明显不通顺的中英文表述
- 对源文件中未独立成章但可由现有内容支持的模块，允许整理性补写成正式章节内容

不要做这些事：

- 编造技术参数
- 无依据补充性能承诺
- 擅自删除可能具有合规意义的限制、警示、储运条件
- 为了“好看”改动实质技术含义

### Redundant Content

如果某句只是重复性的质量背书或泛化表态，对技术使用没有新增信息，例如：

- “经过严格质量控制”
- “符合相关标准”
- “进行了多项性能测试”

且全文别处已有实质信息覆盖，可在不改变合规含义的前提下删除或合并，使说明书更简洁。

### Missing Or Derived Modules

模块缺失时按下面规则处理：

- 源文件没有独立出现、且没有足够内容支撑的模块：直接不出现在成稿里
- `推荐工艺参数 / Recommended Process Parameters` 如果源文件没有独立内容支撑，不要根据零散信息自行补写成完整章节
- 不要在成稿正文中写“源文件未提供某模块”这类说明
- 如需保留风险控制或合规提示模块，例如“重要声明”，可依据全文已有测试说明、适用性验证提醒、责任边界类内容，整理为正式模块
- 这类整理性补写不需要在成稿正文中额外声明“这是补充内容”
- 如需留痕，应单独输出优化报告，而不是把元说明写进说明书正文
- 如果删掉了某个缺失章节，后续章节编号和中英标题序号应自动顺延，不要保留跳号

### Bilingual Style

英文应偏工业产品说明书风格：

- 简洁
- 稳定
- 术语统一
- 单位准确
- 避免营销腔

### Terminology Consistency

同一份说明书中术语必须统一。例如：

- 产品描述 / Product Description
- 应用领域 / Application Fields
- 主要技术指标 / Key Technical Specifications
- 推荐工艺参数 / Recommended Process Parameters
- 使用注意事项 / Precautions for Use

如果源文档出现多个译法，选择一个最稳定、最专业的译法后全篇统一。

## Missing Information Policy

当信息不完整时，优先显式暴露缺口，而不是偷偷略过。

建议处理方式：

- 中文有、英文无：按用户要求补英文，或标注待补
- 有章节标题、无内容：标注 `未提供 / Not Provided`
- 有参数名、无数值：保留参数名并标注 `未提供 / Not Provided`
- 有测试项、无标准：填 `—`
- 有页眉字段位、无公司信息：保留占位

## Final Check

输出前检查：

- 是否保留了统一的 7 章结构
- 是否保持中英顺序一致
- 是否将技术指标收敛成统一表格
- 是否显式标注缺失信息
- 是否避免编造参数
- 是否清理了空 bullet、空编号、空标题、残留空白
- 是否删除了重复应用列表或重复背书内容
- 是否优化了明显不顺的中英文表述
- 是否删除了空项目符号及其整行占位
- 是否避免在正文中出现“源文件未提供”“以下为补充”等元说明
- 若源文件没有“重要声明”，是否根据现有内容判断是否应补成正式模块
- 是否确保表格中中文在上、英文在下，且左对齐
- 是否尽量让表格整页显示，不要只剩表头或被难看拆页
- 是否只保留必要空行
- 是否保证页眉单行可读，logo 贴近公司名但不遮挡文字
